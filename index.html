The problem

RF data generation is the bottleneck

Developing a wearable ultrasound patch means designing custom transducer arrays, tuning beamforming parameters, and validating image quality — all before a physical prototype is ready. Each of these steps depends on realistic synthetic RF (radio-frequency) data that models how acoustic waves propagate through tissue, scatter off structures, and return to the array.

Traditional simulation tools like MATLAB-based MUST run this on CPU and take 2+ hours for a single frame with 50,000 scatterers across 64 transmissions. That makes iterative design practically impossible: change one parameter and wait two hours to see the result.

CardioSim replaces that bottleneck with a JAX/XLA GPU-accelerated engine that generates the same RF data in ~236 seconds and reconstructs the B-mode image in under a second — a ~31× end-to-end speedup. Researchers can now iterate in minutes, not days.

For researchers

Rapidly prototype array configurations, test diverging-wave sequences, and benchmark beamforming algorithms without waiting for hardware or enduring multi-hour CPU simulations. Compress the design–simulate–evaluate loop from days to minutes.

For clinicians

The end goal is a wearable patch that gives doctors continuous cardiac imaging at the bedside — no bulky cart, no trained sonographer holding a probe. CardioSim validates that the patch can produce diagnostic-quality images before it reaches a patient.

01

Pipeline

01
phantom

Scatterer generation

Digital cardiac phantoms converted into intensity-weighted acoustic scatterer maps — up to 50,000 scatterers across a 10 cm × 10 cm field of view.

02
transmit

RF data simulation

64 diverging-wave transmissions at ±45° from a virtual source at −4 cm depth. JAX/XLA on GPU generates raw channel-domain RF at 10.8 MHz sampling — the step that used to take hours.

03
reconstruct

Beamforming & imaging

Delay-and-Sum beamforming with Hanning receive apodisation. Coherent compounding, envelope detection, and 60 dB log-compression to produce the final B-mode sector image.

02

Technical stack

JAX / XLA
GPU-native RF simulation via jit() and vmap(). The core engine behind the ~31× speedup in RF data generation.
UltraSpy
DAS beamforming engine with GPU/CPU fallback, polar scan geometry, and envelope detection.
MATLAB / MUST
Reference SIR-based simulation for cross-platform validation and CPU benchmarking baseline.
PyMUST
Standalone Python reimplementation of MUST — the CPU baseline that JAX replaces.
FPGA back-end
Disaggregated hardware architecture with flexible piezoelectric array on PDMS substrate.
OpenCV / SciPy
Image processing, bandpass filtering, and quality metrics — SSIM, SNR, CNR, FWHM.

Research context

Programme
PhD in Biomedical Engineering at the Singapore Institute of Technology (SIT), supervised by A/Prof Mahesh Raveendranatha Panicker.
Scope
128-channel wearable multi-view echocardiography patch with a disaggregated architecture: flexible piezoelectric array on PDMS substrate, FPGA back-end for data acquisition, and laptop GPU for real-time image reconstruction.
